Giám sát session logs

Tự động cô lập EC2 Instance với AWS Lambda và GuardDuty

Kho lưu trữ này cung cấp hướng dẫn chi tiết để thiết lập quy trình làm việc tự động nhằm cô lập các EC2 instance bị xâm phạm dựa trên các phát hiện từ AWS GuardDuty. Quy trình này sử dụng EC2, GuardDuty, S3, EventBridge và Lambda để phát hiện, xử lý và thông báo về các mối đe dọa bảo mật, đồng thời tuân thủ nguyên tắc quyền tối thiểu cần thiết cho các IAM Role và Policy.

Tổng quan dự án

Dự án này tự động hóa quy trình phát hiện và cô lập các EC2 instance bị xâm phạm theo thời gian thực, giúp giảm thiểu rủi ro bảo mật. Nó được thiết kế cho mọi đối tượng sử dụng AWS, kể cả người mới bắt đầu, và bao gồm các hướng dẫn từng bước để triển khai các tài nguyên và cấu hình cần thiết.

Tính năng:

  • Phát hiện mối đe dọa theo thời gian thực: Sử dụng Amazon GuardDuty để theo dõi các hoạt động độc hại.
  • Tự động cô lập Instance: Dùng AWS Lambda để dừng và cô lập các instance bị xâm nhập.
  • Quy trình linh hoạt: Tích hợp danh sách đe dọa tùy chỉnh thông qua S3.
  • Thông báo: Nhận thông báo thời gian thực qua SNS.

Hướng dẫn thiết lập

1. Tạo IAM Role

Mục đích: Cho phép Lambda quản lý security group, dừng instance và tương tác với GuardDuty.

  1. Vào IAM Console → Roles → Create Role
  2. Chọn AWS Service → Lambda
  3. Gán các policy sau:
    • AmazonEC2FullAccess
    • AWSLambdaFullAccess
  4. Đặt tên (ví dụ: LambdaSecurityRole) và tạo role

2. Kích hoạt GuardDuty

Mục đích: Theo dõi hoạt động độc hại trong môi trường AWS của bạn.

  1. Mở GuardDuty Console
  2. Click Enable GuardDuty nếu chưa bật

3. Khởi tạo hai EC2 Instances

Mục đích: Mô phỏng một kẻ tấn công (MaliciousEC2) và một instance bị xâm nhập (CompromisedEC2).

  1. Vào EC2 Console, chọn Launch Instance
  2. Cấu hình instance:
    • AMI: Amazon Linux 2
    • Type: t2.micro
    • User Data Script: 
      #!/bin/bash
yum update -y
yum install nmap -y
  3. Đặt tên là MaliciousEC2 và CompromisedEC2

4. Tạo Security Group giới hạn

Mục đích: Giới hạn lưu lượng nhằm cô lập các instance bị xâm phạm.

  1. Trong phần Security Groups, tạo một nhóm mới:
    • Tên: IsolatedSecurityGroup
    • Inbound: chỉ cho phép SSH (Port 22) từ IP của bạn
    • Outbound: mặc định

5. Thêm danh sách đe dọa vào S3 và cấu hình GuardDuty

Mục đích: Mô phỏng mối đe dọa bằng cách sử dụng danh sách IP lưu trên S3.

  1. Tạo một S3 Bucket (ví dụ: threat-list-bucket)
  2. Upload file threatlist.txt chứa IP của MaliciousEC2
  3. Trong GuardDuty:
    • Vào Settings → Threat Lists
    • Thêm URL của file threat list trong S3

6. Thiết lập Lambda Function

Mục đích: Tự động xử lý các phát hiện từ GuardDuty.

  1. Tạo function IsolateInstance với runtime Python 3.8+
  2. Thêm mã Python
  3. Gán LambdaSecurityRole cho function
  4. Triển khai và kiểm thử

7. Cấu hình EventBridge Rule

Mục đích: Kích hoạt Lambda khi có phát hiện từ GuardDuty.

  1. Tạo rule trong EventBridge:
    • Tên: GuardDutyToLambdaRule
    • Nguồn sự kiện: GuardDuty
    • Sử dụng event pattern JSON
    • Mục tiêu: Lambda IsolateInstance

8. Tạo SNS Topic để thông báo

Mục đích: Gửi thông báo về sự kiện bảo mật.

  1. Tạo SNS Topic GuardDutyNotifications
  2. Đăng ký email của bạn và xác nhận qua email

Kiểm thử quy trình

  1. Kết nối vào CompromisedEC2 qua SSH
  2. Thực thi script mô phỏng hoạt động độc hại: test.sh
  3. Kiểm tra:
    • Các phát hiện từ GuardDuty
    • Việc instance bị dừng và cô lập
    • Nhận thông báo từ SNS

Dọn dẹp

  1. Xóa tất cả tài nguyên:
    • EC2 instances
    • Lambda functions
    • S3 bucket
    • IAM roles
    • EventBridge rules
    • SNS topics
  2. Đảm bảo không còn tài nguyên để tránh tốn chi phí

Lưu ý thực tế

  1. Tính sẵn sàng của Instance: Dựa vào mức độ nghiêm trọng để chọn cách cô lập, tránh ảnh hưởng không cần thiết.
  2. Giảm cảnh báo sai: Tinh chỉnh GuardDuty và dùng suppression rules.
  3. Thực hành bảo mật nâng cao: Kích hoạt logging toàn diện và bảo mật đa lớp.
